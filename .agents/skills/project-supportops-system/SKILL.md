# Project SupportOps System

Use this when a project change could create user questions.

## Check

1. What will users ask?
2. Which users are affected?
3. Is there a self-service answer?
4. Does support need a macro?
5. Is there an escalation path?
6. Does wording need to be safer/clearer?
7. Does the product need to reduce support load?

## Support Artifacts

Create or update:
- FAQ entry;
- support macro;
- escalation rule;
- known issue note;
- release note;
- internal troubleshooting steps.

## Metrics

Recommend metrics when relevant:
- first response time;
- resolution time;
- reopen rate;
- CSAT;
- deflection rate;
- top issue categories.

## Output

- user confusion risks;
- support docs to update;
- macros/escalations;
- product improvement suggestion.
