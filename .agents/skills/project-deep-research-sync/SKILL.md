# Project Deep Research Sync

This skill prevents shallow work.

## Workflow

1. Scope classification:
   Patch / Polish / Rebuild / System / Safety

2. Ownership Scan:
   - direct impact
   - indirect impact
   - docs/memory impact
   - calendar/roadmap impact
   - legal/business/pricing/support impact
   - QA/release impact
   - decision/source-of-truth impact

3. Role-specific research:
   Each relevant manager must:
   - read local source of truth;
   - search the internet unless explicitly offline/read-only/trivial;
   - bring 2-5 domain findings;
   - include source links for external claims;
   - propose one improvement not explicitly requested.

4. Cross-synthesis:
   - compare findings;
   - identify conflicts;
   - merge into one direction;
   - decide Keep / Remove / Merge / Move / Rename / Redesign.

5. Second-pass research:
   Run if the merged plan introduces a new question around UX, legal, pricing, support, architecture, AI/tool risk, release, or partners.

6. Impact Sync:
   - product/artifact
   - code/data
   - docs/memory
   - calendar/roadmap
   - legal/privacy
   - billing/pricing
   - business/strategy
   - support/customer ops
   - suppliers/partners
   - knowledge/decisions
   - QA/release
