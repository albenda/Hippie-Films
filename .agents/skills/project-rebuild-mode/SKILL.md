# Project Rebuild Mode

Use this when the owner wants a real rebuild, not a patch.

## Required Inventory

List every:
- section
- tab
- CTA/button
- modal/sheet
- data source
- state: empty/loading/error/full
- navigation path
- risky dependency

## Decide For Each Item

Keep
Remove
Merge
Move
Rename
Redesign
Defer

## Rebuild Output

Before code or execution, produce:
1. Current state summary.
2. New information architecture.
3. Wireframe or workflow.
4. CTA hierarchy.
5. Empty/loading/error/full states.
6. Mobile/responsive behavior.
7. What remains unchanged and why.
8. Risks/hard stops.

## Quality Bar

The work fails if:
- it still feels like the same thing with nicer boxes;
- only colors/spacing changed;
- CTAs remain unclear;
- empty states are missing;
- navigation is still confusing;
- no before/after product difference exists.
