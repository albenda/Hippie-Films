# Owner Language Model

מטרה:
ללמד את המערכת להבין את שפת הבעלים ולא לחכות לפרומפט מושלם.

## פירוש ביטויים

| כשהבעלים אומר | המשמעות התפעולית |
|---|---|
| "תסדר לי הכל כמו שצריך" | System Mode או Rebuild Mode. לא תיקון נקודתי. |
| "הכל, הכל, הכל" | Inventory מלא: קבצים, מסכים, כפתורים, מצבים, דאטה, מסמכים והשפעות. |
| "זה עדיין אותו דבר" | העבודה נכשלה מבחינת תפיסה. צריך שינוי מבני, לא polish. |
| "אני רוצה שתבנה מחדש" | Rebuild Mode: היררכיה חדשה, flow חדש, CTAs חדשים, states חדשים. |
| "תשתמש בכל הסוכנים" | Ownership Scan: כל מנהל רלוונטי עושה את חלקו. |
| "תחפש באינטרנט" | Deep Research Sync: מקומי + אינטרנט + סינתזה + Impact Sync. |
| "לא לגעת כרגע" | Read-only. סריקה/ניתוח בלבד. אין עריכה. |
| "יאללה בוא נעבוד" | לבצע בפועל אם אין Hard Stop. |
| "רגע" / "לא לזה התכוונתי" | לעצור, לסכם הבנה מחדש, לא להמשיך לערוך. |
| "תעשה מה שאתה חושב" | חופש פעולה בתוך גבולות Hard Stops ומקור אמת. |

## כללי תגובה

1. אם הבקשה נשמעת רחבה, אל תחזיר תיקון קטן.
2. אם חסר מידע אבל אפשר לגלות מהקבצים, קודם תסרוק.
3. אם יש סיכון משפטי, כסף, פרטיות, דאטה, הרשאות, פרסום או מחיקה — עצור לאישור.
4. אם זה עיצוב רחב — חובה לקרוא את reference design של הפרויקט ולבנות היררכיה חדשה.
5. אם זו משימת תהליך — עדכן את המסמכים, לא רק תענה בשיחה.

## Definition of Done

משימה משמעותית סגורה רק כאשר:
- בוצע source scan.
- הופעלו המנהלים הרלוונטיים.
- נבדקו השפעות רוחביות.
- עודכנו task_board/current_state/interaction_log לפי הצורך.
- נאמר מה נבדק ומה לא נבדק.
